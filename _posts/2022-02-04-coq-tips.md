# Coq이란?
프로그램이 올바르게 작성되었다는 것을 검증하는 방식으로 여러가지가 있는데, 대표적인 세가지로 [model checking](https://en.wikipedia.org/wiki/Model_checking), [interactive theorem proving](https://en.wikipedia.org/wiki/Proof_assistant), 그리고 [static analysis](https://en.wikipedia.org/wiki/Static_program_analysis)가 있다. 이 방법론들은 각각이 장단점을 갖고 있어서, 어느 하나가 다른 것보다 절대적으로 우월하다고는 할 수 없다. coq이라는 것은 그 중에서 interactive theorem proving(이하 ITP) 동네에서 전형적으로 사용되고 있는 툴, 즉 증명보조기(proof assistant)이다. ITP의 기본 아이디어는 명제<->타입, 증명<->프로그램으로 대표되는 대응관계인 소위 [curry-howard isomorphism](https://en.wikipedia.org/wiki/Curry%E2%80%93Howard_correspondence)에 근거한다. 이것이 뭐냐면, 우리가 수학에서 어떤 "명제"를 "증명"한다는 것은, 그야말로 일개 프로그래머가 주어진 "타입"에 들어맞는 적당한 "프로그램"을 작성하는 것과 동등한 행위라는 것이다. 그렇다면 이걸 어떻게 프로그램 검증에 이용할 수 있느냐? 우리가 원하는 성질(가령, "프로그램 add에 인자로 a, b를 넣으면 그 출력값 c는 a+b이다" 따위)을 "타입"으로 인코딩한 후, 그 타입에 해당하는 coq "프로그램"을 아무거나 작성하면 된다. curry-howard isomorphism에 의하면, 이는 "명제"를 "증명"한 셈이기 때문이다. 이런게 가능하기 위해서는 당연히 "타입"이라는 것이 c언어의 그것처럼 단순해서는 안되고, "for all ~", "there exists ~" 따위를 표현할 수 있을 정도로 강력해야 한다. 그러한 타입시스템의 대표적인 예로, 스웨덴의 논리학자인 마틴 뢰프가 지대한 공헌을 한 [dependent types](https://en.wikipedia.org/wiki/Dependent_type)가 있다. 내가 이해하기로는, 프랑스의 전산학자인 Thierry Coquand이 이를 dependent types -> Calculus of Constructions(CoC) -> Calculus of Inductive Constructions(CiC) -> Gallina -> Coq 순으로 확장한 것이 바로 Coq이다 [(참고)](https://www.cs.cornell.edu/courses/cs6120/2022sp/blog/coc/). 실제로 Coq을 이용하여 그 유명한 [4색 정리](https://en.wikipedia.org/wiki/Four_color_theorem)를 증명하기도 했고, [CompCert](https://compcert.org/compcert-C.html)라고 최적화가 올바르다고 수학적으로 검증된 컴파일러가 만들어지기도 했을만큼, Coq은 단순한 연구거리에 불과한 것이 아니라 실제로 쓸모가 있다는 것이 점차 밝혀지고 있다. 가장 낙천적인 관점에서, Coq은 미래에 수학자들의 펜과 종이를 대체할 수 있을 것이다. Curry-Howard Isomorphism에 의하면 수학자들은 이미 이론적으로 "일종의" 프로그래머이지만, Coq(혹은 그런 류의 어느 기타 증명보조기라도)이 수학자들의 원자재가 되는 날에는 수학자들은 "말 그대로" 프로그래머일 것이다. 실제로 러시아의 필즈상 수상자 Voevodsky에 의해 수학의 토대를 프로그래밍의 토대로 갈아엎는 [프로젝트](https://en.wikipedia.org/wiki/Univalent_foundations)가 진행 중이다.

# 셋업

만약 [온라인](https://coq.vercel.app/scratchpad.html)에서만 Coq을 사용할 것이라면, 아래 셋업 단계는 스킵해도 된다.

## 설치
linux 기준으로 설명한다. coq을 커맨드라인 환경에서 이용할 경우 ocaml이라는 언어의 패키지 매니저인 opam으로 설치하면 편한데, coq 자체가 ocaml로 개발됐기 때문이다.
자세한 설치 방법은 [공식 홈페이지](https://coq.inria.fr/opam-using.html)을 참고하면 된다.
시스템에 opam만 있으면 다음처럼 손쉽게 coq을 설치할 수 있다. (참고로, opam에서 coq 소스를 갖고와서 빌드하기 때문에 시간이 좀 걸린다.)

<h5 a><strong><code>shell</code></strong></h5>

```
$ opam pin add coq 8.12.0
$ coqc -v
>> The Coq Proof Assistant, version 8.12.0 (September 2022)
```

위처럼 coqc -v를 시스템에 입력했을 때 뭐가 그럴 듯하게 뜨면 coq이 제대로 설치된 것이다.

## 개발환경

vim에서 coq을 사용할 경우, [coqtail](https://github.com/whonore/Coqtail) 플러그인을 이용하면 된다. 자세한 설치방법은 coqtail 레포지토리를 참고하면 된다. 나는 [vim-plug](https://github.com/junegunn/vim-plug)라는 vim 플러그인 매니저를 이용하여 설치하였다.

우선 vim-plug가 있다고 가정하고, .vimrc 파일의 적당한 곳에 다음을 추가시킨다.
<h5 a><strong><code>~/.vimrc</code></strong></h5>

```
call plug#begin('~/.vim/plugged')
Plug 'whonore/Coqtail'
call plug#end()
```
그런 후에, shell에 다음을 입력하면 설치가 완료된다.
<h5 a><strong><code>shell</code></strong></h5>

```
vim +PlugInstall +qa
```
만약 이 과정에서 문제가 있다면, vim이 python3을 지원하지 않아서일 가능성이 있다. 이 경우 vim을 python3을 지원하도록 다시 인스톨해보자.

coqtail 플러그인이 잘 설치됐는지 확인하기 위해, 임시 coq파일(.v 확장자)를 vim으로 실행시킨 후,
```
$ vim hello.v
```
\cc를 눌러보자. "\\", "c", "c" 키를 순서대로 세번에 걸쳐 연달아 치면된다. 그랬을때 vim 창이 세개로 분리되면서 우하단의 창에 와인잔 비슷한 그림이 나오면 coqtail이 성공적으로 설치된 것이다.

## coq 시작하기
vim 커맨드:
- \cc coq 시작
- \cl 현재 커서까지 coq에 입력
- \cj 한 sentence 추가로 coq에 입력
- \ck 한 sentence coq에서 삭제
- :Coq About <id> <id>의 타입 출력
- :Coq Print <id> <id>의 정의 출력
- :Coq Search <id> <id>를 포함하는 theorem들 목록 출력
* 위 세개는 Coq Query로, ++같은 notation을 검색하려면 "++"와 같이 쌍따옴표로 감싸줘야 한다.
